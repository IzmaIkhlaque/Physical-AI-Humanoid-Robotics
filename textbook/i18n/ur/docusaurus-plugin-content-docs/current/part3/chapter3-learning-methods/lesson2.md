# ہیومنائڈز کے لیے غیر نگرانی شدہ لرننگ: پیٹرن کی دریافت اور بے ضابطگی کا پتہ لگانا

## خلاصہ

*   **سبق 1 - سپروائزڈ لرننگ:** ادراک اور پیشن گوئی کے کام انجام دینے کے لیے لیبل شدہ ڈیٹا سے سیکھنا۔

جبکہ سپروائزڈ لرننگ واضح ان پٹ-آؤٹ پٹ تعلقات والے کاموں کے لیے طاقتور ہے، ہیومنائڈز کے لیے ایک اہم چیلنج بغیر لیبل شدہ حسی ڈیٹا کی بھاری مقدار ہے جس کا وہ سامنا کرتے ہیں۔ حقیقی دنیا میں، ہر معلومات کو احتیاط سے لیبل کرنا اکثر غیر عملی یا ناممکن ہے۔ یہیں پر **غیر نگرانی شدہ لرننگ** انمول بن جاتی ہے۔ غیر نگرانی شدہ لرننگ الگورتھم واضح لیبلز کے بغیر ڈیٹا کے اندر پیٹرن، ڈھانچے، یا تعلقات دریافت کرتے ہیں، جو انہیں ہیومنائڈ روبوٹکس میں ڈیٹا کمپریشن، کلسٹرنگ، اور بے ضابطگی کے پتہ لگانے جیسے کاموں کے لیے مثالی بناتا ہے۔

### 1. غیر نگرانی شدہ لرننگ کی بنیادی باتیں

سپروائزڈ لرننگ کے برعکس، غیر نگرانی شدہ لرننگ بغیر لیبل شدہ ڈیٹا کے ساتھ کام کرتی ہے۔ مقصد ڈیٹا کے اندر چھپے ہوئے ڈھانچے یا نمائندگیوں کو ظاہر کرنا ہے۔

#### اہم تصورات:

*   **کوئی لیبل نہیں:** ٹریننگ ڈیٹا صرف ان پٹ فیچرز (X) پر مشتمل ہوتا ہے، بغیر متعلقہ آؤٹ پٹ لیبلز کے۔
*   **پیٹرن کی دریافت:** الگورتھم ڈیٹا کے اندر موروثی گروپنگ، کثافت، یا جہتوں کی تلاش کرتے ہیں۔
*   **نمائندگی کی تعلیم:** ان پٹ ڈیٹا کی ایک نئی، اکثر کم جہتی، نمائندگی سیکھنا جو اس کی سب سے اہم خصوصیات کو پکڑتی ہے۔

#### عام غیر نگرانی شدہ لرننگ کے کام:

*   **کلسٹرنگ:** ایک جیسے ڈیٹا پوائنٹس کو ایک ساتھ گروپ کرنا (مثلاً، مختلف اشیاء یا ماحولیاتی خصوصیات کی شناخت)۔
*   **ڈائمینشنلٹی ریڈکشن:** ڈیٹاسیٹ میں فیچرز کی تعداد کو کم کرنا جبکہ زیادہ سے زیادہ معلومات کو برقرار رکھنا (مثلاً، اعلیٰ جہتی سینسر ڈیٹا کو کمپریس کرنا)۔
*   **بے ضابطگی کا پتہ لگانا:** ڈیٹا پوائنٹس کی شناخت کرنا جو عام سے نمایاں طور پر منحرف ہیں (مثلاً، غیر معمولی سینسر ریڈنگز یا روبوٹ کے رویے کا پتہ لگانا)۔

### 2. ہیومنائڈ ادراک کے لیے غیر نگرانی شدہ لرننگ

غیر نگرانی شدہ طریقے بڑے لیبل شدہ ڈیٹاسیٹس کی ضرورت کے بغیر ہیومنائڈ کے ادراک کو بہتر بنا سکتے ہیں:

*   **فیچر نکالنا/نمائندگی کی تعلیم:** Autoencoders اور Principal Component Analysis (PCA) اعلیٰ جہتی سینسر ڈیٹا (مثلاً، خام کیمرہ تصاویر، lidar اسکین) کی جامع اور بامعنی نمائندگیاں سیکھ سکتے ہیں۔ یہ سیکھی ہوئی خصوصیات پھر ذیلی سپروائزڈ کاموں میں یا موثر ذخیرہ کرنے کے لیے استعمال کی جا سکتی ہیں۔
*   **ماحولیاتی نقشہ سازی:** کلسٹرنگ الگورتھم خام سینسر ڈیٹا سے ماحول میں مختلف خصوصیات کی شناخت کر سکتے ہیں، جو روبوٹ کو پہلے سے طے شدہ لیبلز کے بغیر نقشے بنانے یا علاقوں کی درجہ بندی کرنے میں مدد کرتے ہیں۔
*   **آبجیکٹ سیگمنٹیشن:** اگرچہ اکثر سپروائزڈ طریقوں سے انجام دیا جاتا ہے، غیر نگرانی شدہ تکنیکیں اشیاء کو ان کے پس منظر سے الگ کرنے میں مدد کر سکتی ہیں جیسی خصوصیات (مثلاً، رنگ، بناوٹ) والے علاقے تلاش کر کے۔

### 3. ہیومنائڈ رویے اور بے ضابطگی کے پتہ لگانے کے لیے غیر نگرانی شدہ لرننگ

غیر نگرانی شدہ لرننگ ہیومنائڈ کے اپنے رویے کو سمجھنے اور نگرانی کرنے اور غیر متوقع واقعات کا پتہ لگانے کے لیے بھی اہم ہے:

*   **سینسر ڈیٹا میں بے ضابطگی کا پتہ لگانا:** خراب سینسرز، غیر متوقع بیرونی قوتوں، یا غیر معمولی ماحولیاتی حالات کا پتہ لگانا۔ مثال کے طور پر، ایک روبوٹ چلتے وقت جوائنٹ ٹارکس کی "عام" رینج سیکھ سکتا ہے؛ کوئی بھی اہم انحراف مسئلے کی نشاندہی کر سکتا ہے۔
*   **خرابی کا پتہ لگانا اور تشخیص:** ایسے پیٹرن کی شناخت کرنا جو مکینیکل گھسائی، موٹر کی ناکامیوں، یا دیگر نظام کی خرابیوں کی نشاندہی کرتے ہیں۔ یہ پیشن گوئی کی دیکھ بھال کی اجازت دیتا ہے۔
*   **تعامل کے پیٹرن کی دریافت:** انسان-روبوٹ تعامل میں، مشاہدہ شدہ انسانی اعمال کی کلسٹرنگ عام تعامل کے پیٹرن کو ظاہر کر سکتی ہے یا مختلف انسانی صارفین کو ان کے تعامل کے انداز کی بنیاد پر گروپ کر سکتی ہے۔
*   **مہارت کی دریافت (ہائبرڈ RL/غیر نگرانی شدہ):** کچھ جدید RL تکنیکیں غیر نگرانی شدہ لرننگ کو جوڑتی ہیں تاکہ واضح انعامات کے بغیر مفید ذیلی اہداف یا مہارتیں دریافت کی جائیں، جو سیکھنے کے عمل کو زیادہ موثر بناتی ہیں۔

**کوڈ اسنیپٹ مثال (تصوراتی K-Means کلسٹرنگ):**

```python
import numpy as np
from sklearn.cluster import KMeans
import matplotlib.pyplot as plt

# --- 1. Simulate Humanoid Sensor Data ---
# Imagine a robot collecting 2D data points representing, e.g.,
# (distance_to_object, color_intensity) for various observed items.
# We want to find natural groupings (clusters) without knowing the labels.

# Cluster 1: Close, bright objects (e.g., small, reflective items)
np.random.seed(0) # for reproducibility
data_cluster1 = np.random.normal(loc=[1.0, 0.8], scale=[0.2, 0.1], size=(50, 2))
# Cluster 2: Far, dim objects (e.g., distant furniture)
data_cluster2 = np.random.normal(loc=[5.0, 0.2], scale=[0.5, 0.15], size=(50, 2))
# Cluster 3: Mid-range, moderately bright objects (e.g., humans)
data_cluster3 = np.random.normal(loc=[2.5, 0.5], scale=[0.3, 0.08], size=(50, 2))

X = np.vstack((data_cluster1, data_cluster2, data_cluster3))

# --- 2. Apply K-Means Clustering ---
# Let's assume the robot is looking for 3 distinct types of environmental features.
num_clusters = 3
kmeans = KMeans(n_clusters=num_clusters, random_state=0, n_init=10) # n_init for robust centroid initialization
kmeans.fit(X)

# --- 3. Visualize Results (Conceptual) ---
labels = kmeans.labels_
centroids = kmeans.cluster_centers_

# plt.figure(figsize=(8, 6))
# plt.scatter(X[:, 0], X[:, 1], c=labels, cmap='viridis', s=50, alpha=0.7, label='Data Points')
# plt.scatter(centroids[:, 0], centroids[:, 1], c='red', marker='X', s=200, label='Centroids')
# plt.title('K-Means Clustering of Humanoid Sensor Data')
# plt.xlabel('Distance to Object')
# plt.ylabel('Color Intensity')
# plt.legend()
# plt.grid(True)
# plt.show()

print(f"کلسٹر لیبلز: {labels[:10]}...")  # پہلے 10 ڈیٹا پوائنٹس
print(f"کلسٹر سینٹروئڈز:\n{centroids}")
print("غیر نگرانی شدہ لرننگ نے 3 مختلف قسم کی ماحولیاتی خصوصیات دریافت کیں!")
```

### ہیومنائڈز میں غیر نگرانی شدہ لرننگ کے چیلنجز

*   **کلسٹرز کی تعداد کا تعین:** K-Means جیسے الگورتھم کے لیے، کلسٹرز کی تعداد کا پتہ لگانا (K) خود ایک چیلنج ہو سکتا ہے۔
*   **تشریح:** خودکار طور پر دریافت شدہ کلسٹرز یا فیچرز کی تشریح ہمیشہ آسان نہیں ہوتی۔
*   **اسکیل ایبلٹی:** کچھ غیر نگرانی شدہ طریقے بڑے، اعلیٰ جہتی ڈیٹاسیٹس کے ساتھ کمپیوٹیشنل لحاظ سے مہنگے ہو سکتے ہیں۔
*   **بے ضابطگی کے پتہ لگانے میں جھوٹے مثبت/منفی:** "عام" کی حد کو بہت سخت یا بہت ڈھیلا مقرر کرنا غلط الارمز یا چھوٹی ہوئی اصل بے ضابطگیوں کا باعث بن سکتا ہے۔

### سرگرمیاں

1.  **کلسٹرنگ ورزش:** تصور کریں کہ ایک ہیومنائڈ ایک کمرے میں متعدد اشیاء دیکھتا ہے (کتابیں، کھلونے، برتن)۔ روبوٹ ان اشیاء کو پہلے سے طے شدہ لیبلز کے بغیر "قسموں" میں کیسے گروپ کر سکتا ہے (K-Means کا استعمال کرتے ہوئے)؟ یہ گروپنگ کن خصوصیات پر مبنی ہو سکتی ہے (شکل، سائز، رنگ)؟

2.  **بے ضابطگی کا پتہ لگانا منظر نامہ:** ایک ایسے منظر نامے کی وضاحت کریں جہاں ایک ہیومنائڈ روبوٹ کے لیے اپنے جوائنٹ موٹرز میں بے ضابطگی کا پتہ لگانا اہم ہو گا (مثلاً، ایک موٹر زیادہ گرم ہو رہی ہے، ایک جوائنٹ عام سے زیادہ مزاحمت ظاہر کر رہا ہے)۔ روبوٹ اس قسم کی بے ضابطگیوں کا پتہ لگانے کے لیے غیر نگرانی شدہ لرننگ کا استعمال کیسے کر سکتا ہے؟
