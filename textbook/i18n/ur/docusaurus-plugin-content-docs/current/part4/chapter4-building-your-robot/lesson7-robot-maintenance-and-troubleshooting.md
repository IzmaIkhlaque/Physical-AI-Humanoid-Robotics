# روبوٹ کی دیکھ بھال اور خرابیوں کی تشخیص

## خلاصہ

*   **سبق 1-6 (اس باب کے پچھلے موضوعات):** (فرض کیا جاتا ہے کہ پچھلے اسباق میں ڈیزائن، تعمیر، پروگرامنگ، جانچ، حفاظت وغیرہ کا احاطہ کیا گیا تھا)

ایک روبوٹ بنانا صرف پہلا قدم ہے؛ اسے فعال، محفوظ اور بہترین کارکردگی کے ساتھ برقرار رکھنے کے لیے مستعد **دیکھ بھال** اور مؤثر **خرابیوں کی تشخیص** کی ضرورت ہوتی ہے۔ کسی بھی پیچیدہ مشین کی طرح، روبوٹ بھی ٹوٹ پھوٹ، سافٹ ویئر کی خرابیوں اور غیر متوقع ناکامیوں کا سامنا کرتے ہیں۔ اپنے روبوٹ کی منظم طریقے سے دیکھ بھال کرنے اور مسائل کی تشخیص کرنے کا طریقہ سمجھنا اس کی عمر بڑھانے اور اس کی افادیت کو زیادہ سے زیادہ کرنے کے لیے بہت ضروری ہے۔

### 1. احتیاطی دیکھ بھال کی حکمت عملی

احتیاطی دیکھ بھال کا مقصد روبوٹ کے اجزاء کا باقاعدگی سے معائنہ، صفائی اور سروسنگ کرکے غیر متوقع خرابیوں کو کم کرنا ہے۔

*   **شیڈولڈ معائنہ (Scheduled Inspections):**
    *   **مکینیکل (Mechanical):** ڈھیلے فاسٹنرز، گھسے ہوئے گیئرز، ٹوٹی ہوئی لنکیجز، جوڑوں کی مناسب لبریکیشن کی جانچ کریں۔ کیبلز کو ٹوٹ پھوٹ یا نقصان کے لیے معائنہ کریں۔
    *   **الیکٹریکل (Electrical):** کنکشنز کی تصدیق کریں، وائرنگ کو شارٹ سرکٹ یا ٹوٹ پھوٹ کے لیے معائنہ کریں، بیٹری کی صحت کی جانچ کریں (اگر قابل اطلاق ہو)، موٹر کے درجہ حرارت کی نگرانی کریں۔
    *   **سینسرز (Sensors):** سینسر لینسز (کیمرے، لیڈار) کو صاف کریں، کیلیبریشن کی جانچ کریں، محفوظ ماؤنٹنگ کو یقینی بنائیں۔
*   **صفائی (Cleaning):** دھول، ملبہ اور آلودگی حرکت پذیر حصوں اور سینسرز میں مداخلت کر سکتی ہے۔ باقاعدہ صفائی، خاص طور پر آپٹیکل سینسرز اور کولنگ وینٹ کی، بہت ضروری ہے۔
*   **لبریکیشن (Lubrication):** گیئرز، بیرنگز اور حرکت پذیر جوڑوں پر مینوفیکچرر کی سفارش کے مطابق مناسب چکنائی لگانا تاکہ رگڑ اور ٹوٹ پھوٹ کو کم کیا جا سکے۔
*   **سافٹ ویئر اپ ڈیٹس (Software Updates):** روبوٹ کے آپریٹنگ سسٹم، فرم ویئر اور کنٹرول سافٹ ویئر کو بگز کو ٹھیک کرنے، کارکردگی کو بہتر بنانے اور نئی خصوصیات شامل کرنے کے لیے باقاعدگی سے اپ ڈیٹ کرنا۔
*   **بیک اپ اور ریکوری (Backup and Recovery):** کنفیگریشن فائلوں، کیلیبریشن ڈیٹا اور سیکھے ہوئے ماڈلز کا باقاعدگی سے بیک اپ لینا۔ سافٹ ویئر کی ناکامیوں کے لیے ایک مضبوط ریکوری پلان کا ہونا۔

### 2. روبوٹ کے عام مسائل اور ان کی علامات

| مسئلہ کی قسم             | عام علامات                                                 | ممکنہ وجوہات                                                     |
| :----------------------- | :------------------------------------------------------- | :-------------------------------------------------------------- |
| **مکینیکل مسائل**        | غیر معمولی آوازیں (گھسنے کی، چیخنے کی)، جھٹکے دار حرکتیں، حرکت کی محدود رینج، ظاہری نقصان، جوڑوں کا زیادہ گرم ہونا | گھسے ہوئے گیئرز/بیرنگز، ڈھیلے فاسٹنرز، ناکافی لبریکیشن، موٹر کا اوورلوڈ، جسمانی اثر |
| **الیکٹریکل/پاور**       | روبوٹ کا غیر فعال ہونا، وقفے وقفے سے آپریشن، بجلی کا اتار چڑھاؤ، بیٹری کا چارج نہ ہونا، کنٹرولر پر ایرر لائٹس | ڈھیلی وائرنگ، خراب پاور سپلائی/بیٹری، موٹر ڈرائیور کی خرابی، شارٹ سرکٹ، سینسر کی خرابی |
| **سینسر کے مسائل**       | غلط ریڈنگز، نیویگیشن کی خرابیاں، آبجیکٹ کی شناخت میں ناکامی، روبوٹ کا چیزوں سے ٹکرانا | گندے لینسز، ڈھیلے کنکشنز، سافٹ ویئر کیلیبریشن کے مسائل، سینسر کی ناکامی، بیرونی مداخلت (روشنی، آواز) |
| **سافٹ ویئر/کنٹرول**     | روبوٹ کا کمانڈز پر ردعمل نہ دینا، غیر ارادی حرکتیں کرنا، منجمد ہونا، کنسول پر ایرر میسجز، بار بار ایک ہی عمل | کوڈ میں بگ، خراب فرم ویئر، غلط کنفیگریشن، نیٹ ورک مواصلات کے مسائل، کنٹرولر کا اوورلوڈ |
| **مواصلات کے مسائل**     | تاخیر سے جوابات، گمشدہ کمانڈز، ریموٹ کنٹرول پر روبوٹ کا غیر فعال ہونا، سینسر ڈیٹا کا ضائع ہونا | ڈھیلی کیبلز، وائی فائی کی مداخلت، خراب ٹرانسسیورز، غلط نیٹ ورک سیٹنگز |

### 3. خرابیوں کی تشخیص کا طریقہ کار

خرابیوں کی تشخیص کے لیے ایک منظم طریقہ کار روبوٹ کے مسائل کی تیزی سے شناخت اور حل کرنے میں مدد کر سکتا ہے:

1.  **مشاہدہ اور دستاویز کریں (Observe and Document):** مسئلے کو واضح طور پر بیان کریں۔ کیا ہوا؟ کب؟ کہاں؟ مسئلے سے پہلے روبوٹ کے کیا اقدامات تھے؟ کسی بھی ایرر میسجز یا لاگز کو جمع کریں۔
2.  **مسئلے کو دوبارہ پیدا کریں (Reproduce the Problem):** کیا آپ قابل اعتماد طریقے سے مسئلے کو دوبارہ پیدا کر سکتے ہیں؟ یہ متغیرات کو الگ کرنے میں مدد کرتا ہے۔
3.  **پہلے سادہ چیزوں کی جانچ کریں (Check Simple Things First):**
    *   کیا یہ پلگ ان ہے؟ کیا یہ آن ہے؟
    *   کیا تمام کیبلز محفوظ طریقے سے جڑی ہوئی ہیں؟
    *   کیا کوئی ظاہری رکاوٹیں یا نقصان ہیں؟
    *   کیا ایمرجنسی اسٹاپ بٹن دبا ہوا ہے؟
4.  **مسئلے کو الگ کریں (Isolate the Problem):** مسئلے کو کسی مخصوص ذیلی نظام (جیسے موٹر، موٹر ڈرائیور، سافٹ ویئر ماڈیول) تک محدود کرنے کی کوشش کریں۔
    *   اگر کوئی جوڑ حرکت نہیں کر رہا ہے، تو کیا یہ موٹر ہے، موٹر ڈرائیور ہے، وائرنگ ہے، یا کنٹرولر سے کمانڈ ہے؟
    *   کیا آپ روبوٹ کے دوسرے حصوں کو کنٹرول کر سکتے ہیں؟
5.  **مفروضوں کی جانچ کریں (Test Hypotheses):** اپنے مشاہدات اور علیحدگی کی بنیاد پر، وجہ کے بارے میں ایک مفروضہ بنائیں اور اس کی جانچ کریں۔
    *   "میرے خیال میں بائیں بازو کی موٹر خراب ہے۔" -> موٹر کو ایک معلوم اچھی موٹر سے تبدیل کریں (اگر ممکن ہو)۔
6.  **درستگی کی تصدیق کریں (Verify the Fix):** ممکنہ درستگی کو لاگو کرنے کے بعد، یقینی بنائیں کہ اصل مسئلہ حل ہو گیا ہے اور کوئی نیا مسئلہ پیدا نہیں ہوا ہے۔
7.  **سیکھیں اور روکیں (Learn and Prevent):** مسئلے، اس کی وجہ اور اس کے حل کو دستاویز کریں۔ دوبارہ ہونے سے روکنے کے لیے دیکھ بھال کے معمولات یا ڈیزائن کو اپ ڈیٹ کریں۔

**کوڈ کا نمونہ مثال (تصوراتی خود تشخیصی جانچ):**

```python
import time
import random

class RobotDiagnostic:
    def __init__(self, robot_name):
        self.robot_name = robot_name
        self.sensor_status = {"camera": "OK", "lidar": "OK", "imu": "OK"}
        self.motor_status = {"left_arm": "OK", "right_arm": "OK", "left_leg": "OK", "right_leg": "OK"}
        self.battery_level = 95
        self.error_log = []

    def run_sensor_self_test(self):
        print(f"{self.robot_name}: Running sensor self-test...")
        for sensor in self.sensor_status:
            if random.random() < 0.05: # 5% chance of failure
                self.sensor_status[sensor] = "ERROR"
                self.error_log.append(f"[{time.time()}] Sensor Error: {sensor} failure detected.")
            else:
                self.sensor_status[sensor] = "OK"
        print("Sensor self-test complete.")

    def run_motor_self_test(self):
        print(f"{self.robot_name}: Running motor self-test...")
        for motor in self.motor_status:
            if random.random() < 0.02: # 2% chance of failure
                self.motor_status[motor] = "WARNING"
                self.error_log.append(f"[{time.time()}] Motor Warning: {motor} showing unusual torque values.")
            else:
                self.motor_status[motor] = "OK"
        print("Motor self-test complete.")

    def check_battery(self):
        self.battery_level -= random.randint(1, 3) # Simulate usage
        if self.battery_level < 20:
            self.error_log.append(f"[{time.time()}] Battery Alert: Low battery level ({self.battery_level}%).")
            return "LOW"
        if self.battery_level < 5:
            self.error_log.append(f"[{time.time()}] Battery Critical: Shutting down soon.")
            return "CRITICAL"
        return "OK"

    def perform_full_diagnostic(self):
        self.error_log = [] # Clear previous errors
        self.run_sensor_self_test()
        self.run_motor_self_test()
        battery_status = self.check_battery()

        print(f"\n--- {self.robot_name} Diagnostic Report ---")
        print("Sensor Status:", self.sensor_status)
        print("Motor Status:", self.motor_status)
        print(f"Battery Level: {self.battery_level}% ({battery_status})")
        
        if self.error_log:
            print("\nErrors/Warnings:")
            for error in self.error_log:
                print(error)
            return "FAIL"
        else:
            print("\nAll systems nominal.")
            return "PASS"
